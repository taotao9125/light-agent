# 🗓 Day 3 —— 并发安全（核心能力）

```txt
[✅] 写时间冲突 SQL
  - pool.getConnection(); 之前这里设计成了单例模式，所有的请求都走一个 connection, 导致失败
  不过这里有点缺陷，connection 是通过函数传到 repository 里的
[✅] 加事务（BEGIN / COMMIT / ROLLBACK）
[✅] 实现 SELECT ... FOR UPDATE
    try {
      await connection.beginTransaction()
      ...所有sql都走同一个connection()
      await connection.commit();
    } catch (e) {
      await connection.rollback();
      throw e;
    } finally {
      connection.release();
    }
    
  
[✅] 加状态字段（pending / confirmed / cancelled）
[✅] 多请求同时预定同一时间段测试
```
