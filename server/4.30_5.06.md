# 🐳 二、Docker 基础操作

## 🔹 查看

```bash
docker ps           # 运行中的容器
docker ps -a        # 所有容器
docker images       # 所有镜像
```

---

## 🔹 构建 / 拉取 / 推送

```bash
# 构建镜像
docker build -t username/meeting-backend:latest .

# 登录 DockerHub
docker login

# 拉取镜像
docker pull username/meeting-backend:latest

# 推送镜像
docker push username/meeting-backend:latest
```

---

## 🔹 运行容器

```bash
docker run -d \
  --name meeting-backend \
  -p 3000:3000 \
  -e DB_HOST=xxx \
  -e DB_PORT=3306 \
  -e DB_USER=root \
  -e DB_PASSWORD=123456 \
  -e DB_NAME=meeting_room_system \
  username/meeting-backend:latest
```

---

## 🔹 查看日志（排错核心）

```bash
docker logs 容器名
```

---

## 🔹 停止 / 删除

```bash
docker stop 容器名
docker rm 容器名
docker rmi 镜像名
```

---

## 🔹 清理

```bash
docker image prune -a
```

# 📦 三、镜像离线搬运（解决无法访问 DockerHub）

```bash
# 本地导出
docker save -o meeting-backend.tar username/meeting-backend:latest

# 上传服务器
scp -i ~/.ssh/xxx.pem meeting-backend.tar ubuntu@IP:~

# 服务器导入
docker load -i meeting-backend.tar
```

# 🚨 八、你踩过的关键坑（必须记住）

```txt
1. docker login 报错 → secrets 没读到
2. docker pull 超时 → 服务器网络问题
3. Redis NaN → env 没配
4. find 查不到 → 用错 -type（d vs f）
5. image 删不掉 → 被 container 占用
6. localhost 连接失败 → 容器内要用服务名
```
