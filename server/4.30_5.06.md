# 🗓 今日问题 & 解决方案

---

## 1️⃣ docker-compose 报错 `'ContainerConfig'`

**问题：**

```txt
docker-compose up 报错 ContainerConfig
```

**原因：**

```txt
docker-compose v1（1.29.2）与新 Docker 不兼容
```

**解决：**

```bash
全部替换为 docker compose（v2）
```

---

## 2️⃣ compose 警告 `version is obsolete`

**问题：**

```txt
version: "3.9" 被提示废弃
```

**原因：**

```txt
Compose v2 已不需要 version 字段
```

**解决：**

```yaml
删除 version 字段
```

---

## 3️⃣ 容器名冲突（redis_meeting_db）

**问题：**

```txt
container name already in use
```

**原因：**

```txt
写了 container_name + 旧容器未清理
```

**解决：**

短期：

```bash
docker rm -f redis_meeting_db
```

长期（关键）：

```txt
删除 container_name，让 compose 自动命名
```

---

## 4️⃣ nginx 无法代理 host 上 backend

**问题：**

```txt
nginx 容器访问不到 host:3000
```

**原因：**

```txt
容器内不能用 localhost 访问宿主机
```

**解决：**

```nginx
proxy_pass http://host.docker.internal:3000;
```

---

## 5️⃣ nginx 未启动

**问题：**

```txt
nginx 没跑起来
```

**原因：**

```txt
实际上跑的是旧容器（带 container_name）
```

**解决：**

```bash
删除旧容器 → 用新 compose 启动
```

---

## 6️⃣ MySQL 初始化 SQL 不执行

**问题：**

```txt
/docker-entrypoint-initdb.d 没生效
```

**原因：**

```txt
volume 已存在，MySQL 不再初始化
```

**解决：**

```bash
docker compose down -v
docker compose up -d
```

---

## 7️⃣ CI 部署导致容器冲突 / 重启服务

**问题：**

```txt
每次部署冲突 / 想 rm 容器
```

**原因：**

```txt
container_name 固定 + 非增量更新
```

**解决（最终方案）：**

```bash
docker compose pull backend
docker compose up -d --no-deps backend
```

---

## 8️⃣ 日志去哪了（不用 PM2）

**问题：**

```txt
没有 pm2，日志怎么看
```

**原因：**

```txt
容器接管 stdout/stderr
```

**解决：**

```bash
docker compose logs -f backend
```

---

## 9️⃣ ab 压测带 token

**问题：**

```txt
接口需要 Authorization
```

**解决：**

```bash
-H "Authorization: Bearer xxx"
```

---

## 🔥 今日核心收获（最重要）

```txt
1. docker compose v2 是标准
2. container_name 是坑（生产建议不用）
3. 容器通信 ≠ localhost（用 service name / host.docker.internal）
4. MySQL 初始化只执行一次（和 volume 强绑定）
5. Docker 已经替代 PM2 的进程管理能力
6. 部署要做“增量更新”，不是 rm 重来
```
