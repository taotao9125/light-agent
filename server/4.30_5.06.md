# 🗓 Day 4 —— GitHub Actions（CI）

```txt
[ ] push 自动 pnpm install
[ ] 自动 pnpm test
[ ] 自动 docker build
```

1. 接入 Nginx
   - 用 80 端口代理 backend:3000
   - 验证 http://公网IP/health 可访问
   - 后端 3000 后续不直接暴露

2. 整理 docker-compose.prod.yaml
   - backend / mysql / redis / nginx
   - env_file
   - restart: unless-stopped
   - depends_on
   - volumes

3. 手动验证生产启动流程
   - docker-compose -f docker-compose.prod.yaml down
   - docker-compose -f docker-compose.prod.yaml up -d --build
   - docker-compose -f docker-compose.prod.yaml ps
   - curl http://localhost/health

4. 写 GitHub Actions 最小 CI/CD
   - push main 触发
   - SSH 登录服务器
   - cd 项目目录
   - git pull
   - docker-compose -f docker-compose.prod.yaml up -d --build

5. 配置 GitHub Secrets
   - SERVER_HOST
   - SERVER_USER
   - SERVER_SSH_KEY
   - SERVER_PORT

6. 做一次真实验证
   - 本地改 /health 返回内容
   - push 到 main
   - GitHub Actions 自动部署
   - 公网访问确认更新

目标：

```txt
代码提交 → 自动验证
```
