# 🗓 Day 1 —— PM2 + 日志（运行体系）

```txt
[✅] pm2 start app.js 成功跑服务
    - 增  pm2 start bin/www --name meeting_room
    - 删  pm2 delete id|name|all
    - 查
      - pm2 list 查看所有应用整体状态
      - pm2 status 查看进程信息
      - pm2 show meet_room 查看进程信息
      - pm2 logs meet_room // Tailing last 15 lines
    - 改
      - pm2 restart meet_room
        - kill old -> start new
      - pm2 reload meet_room
        - start new  -> kill old
      - pm2 stop meet_room
          
[✅] pm2 logs 能看到输出
    - pm2 logs bookstore-server
[✅] 新建 ecosystem.config.js 并启动成功
    这是启动配置文件, 不支持 esm, 只能 .cjs
    -> pm2 start ecosystem.config.cjs

[✅] 配置日志输出到 ./logs/access.log
    通过配置  ecosystem.config.js 
[✅] 配置错误日志到 ./logs/error.log
    ecosystem.config.js 
[✅] 封装 logger（info / error）
    - 最主要的是 code, message, code 是被其他日志服务消费的，message 是给人看的
      错误统一在中间件里收口 ->  logger.error(message, status, {code: 'XXX_NOT_XXX', user_id, room_id, time})
      关键业务打 info ->  logger.info('BOOKING_CREATED', {userId, roomId: room_id, startTime: s, endTime: e});
[✅] 请求接口 → access.log 有记录
[✅] throw error → error.log 有记录
[✅] pm2 restart 后服务正常
```
